# Dashboard v2 — линза, прогресс, прогноз

## 1. Context & Problem

SPEC-013 дал дашборд с 5 KPI и тремя графиками. Но при использовании выявились
две проблемы прозрачности и пользы:

1. **Непрозрачность линзы.** Не видно, учитываются ли в каждой цифре целевые
   фонды (напр. накопления на ипотеку) и goal-помеченные доходы. Net worth
   показывает split «свободно/целевые», Runway — обе версии, а Доход/Норма —
   «всё подряд» без пометки. Жёстко зашитая линза, пользователь путается.
2. **Нет ощущения прогресса и будущего.** KPI показывают только «сейчас».
   Человеку важно видеть, как было раньше (динамика) и куда идёт (прогноз):
   когда закроется цель, какой net worth через год.

## 2. Goals

- **G1 (линза)**: Глобальный тумблер **«Свободные деньги ↔ Со всеми фондами»**
  вверху дашборда, **по умолчанию — «Свободные»**. Переключает линзу разом для
  Net worth, Runway, Доход, Норма сбережений. Траты от линзы не зависят.
- **G2 (подписи)**: Под каждым KPI — короткая подпись, что именно посчитано в
  текущей линзе (снимает «учитывается или нет»).
- **G3 (Δ к прошлому периоду)**: На каждой KPI — стрелка и % изменения к
  предыдущему эквивалентному окну (текущие 3 мес vs предыдущие 3 мес; net worth —
  vs 3 месяца назад). Цвет — «лучше/хуже» по смыслу метрики.
- **G4 (sparkline)**: В каждой KPI-карточке — мини-тренд за окно графиков
  (~12 мес), построенный из существующих series.
- **G5 (проекция net worth)**: На графике «Net worth по времени» — пунктир в
  будущее (6–12 мес) по текущему темпу свободных сбережений.
- **G6 (ETA целей)**: Для каждой активной цели — прогноз даты достижения при
  текущем темпе + сравнение с дедлайном (успеваю / опаздываю на N мес).

## 3. Non-Goals

- **NG1**: Никаких новых таблиц/миграций D1 — только агрегация (как SPEC-013).
- **NG2**: Не трогаем Mini App (CLAUDE.md §11).
- **NG3**: Не делаем сложные прогнозы (ML, сезонность, регрессии) — на личных
  данных это шум. Только линейная экстраполяция по среднему темпу.
- **NG4**: Не делаем настраиваемое окно KPI (1/3/6/12) в этой итерации — окно
  остаётся 3 полных месяца (сглаживание), «движение» показывает Δ. Можно позже.
- **NG5**: Не делаем прогноз трат/дохода по категориям — только агрегатный net
  worth + ETA целей.
- **NG6**: Тумблер линзы — client-side state (без сохранения в БД/настройках
  пользователя). Дефолт «Свободные» при каждой загрузке.

## 4. User journeys

### Happy path
1. Открываю `/` → вверху KPI в линзе **«Свободные»** (дефолт). Net worth = свободные
   деньги (без целевых фондов), Доход = свободный доход, Норма — из него.
2. Под каждой цифрой — подпись («без целевых фондов», «свободный доход» и т.п.)
   и Δ к прошлым 3 мес (↑/↓ %). В карточке — мини-спарклайн тренда.
3. Жму тумблер **«Со всеми фондами»** — Net worth/Runway/Доход/Норма мгновенно
   перестраиваются на полные значения (без рефетча — обе версии в ответе).
4. На графике net worth вижу пунктирную проекцию: «при текущем темпе через год ≈ X».
5. В блоке целей вижу: «Ипотека — при +1000 €/мес закроется к {дата}, дедлайн
   {дата} (успеваю / опаздываю на N мес)».

### Edge cases
- **E1 (нет goal-доходов в окне)**: свободный доход = полный доход; линза не
  меняет цифру — корректно (текущая ситуация Stepan'а).
- **E2 (нет предыдущего окна / 0 данных)**: Δ не вычислим → не показываем бейдж
  (не «−100%»/«∞%»).
- **E3 (темп ≤ 0, проедание)**: проекция net worth — пологая/вниз; ETA целей →
  «при текущем темпе не достигается» (не отрицательная дата).
- **E4 (цель без target_amount или без deadline)**: ETA считаем если есть
  target; нет дедлайна → показываем только прогноз-дату без сравнения. Нет
  target → «цель без суммы» (как сейчас на GoalDetail).
- **E5 (цель уже достигнута, balance ≥ target)**: «достигнута» вместо ETA.
- **E6 (savings_rate отрицательный)**: показываем как есть (тратишь больше, чем
  зарабатываешь) — честный сигнал, не прячем.
- **E7 (деление на ноль)**: prev = 0 в Δ → скрываем Δ; income_free = 0 → норма `—`.

## 5. Data model

**Изменений в D1 нет.** Дополнительно к SPEC-013 читаем `incomes.goal_id`
(уже в схеме) — чтобы отделить goal-помеченные доходы от свободных.

Определения линзы:
- **Свободные** (дефолт):
  - net worth = `free_net_worth_eur` (= total − targeted)
  - доход/мес = `monthly_income_free_eur` (= доход за окно **минус** доходы с `goal_id != null`) / 3
  - норма = `(monthly_income_free − monthly_burn) / monthly_income_free`
  - runway = `free_net_worth / monthly_burn` (= существующий `runway_months`)
- **Со всеми фондами**:
  - net worth = `net_worth_eur` (total)
  - доход/мес = `monthly_income_eur` (все доходы)
  - норма = `savings_rate` (существующий)
  - runway = `net_worth / monthly_burn` (= существующий `runway_months_total`)
- **Траты/мес** — одинаковы в обеих линзах (расход не зависит от назначения дохода).

## 6. API contract — расширение `GET /v1/web/dashboard`

Ответ SPEC-013 сохраняется, в `kpi` добавляются поля:

```jsonc
"kpi": {
  // ... существующие (SPEC-013) ...
  "monthly_income_free_eur": 3100.00,   // доход за окно минус goal-tagged, /3
  "savings_rate_free": 0.25,           // (income_free − burn)/income_free; null если income_free ≤ 0

  // previous-period (для Δ): то же окно, сдвинутое на burn_window_months назад
  "prev_monthly_burn_eur": 2400.00,
  "prev_monthly_income_eur": 3300.00,
  "prev_monthly_income_free_eur": 3300.00,
  "prev_net_worth_eur": 9900.00,        // net worth на конец предыдущего окна (today − 3 мес)
  "prev_free_net_worth_eur": 7700.00
}
```

- **net worth «3 месяца назад»** для Δ: `effective_balance(bucket, endOf(today − 3 мес))` → EUR по `rateAt(той даты)`. Берём из того же in-memory прохода, что и `net_worth_series`.
- Спарклайны строятся на клиенте из `net_worth_series` (net worth) и
  `cashflow_series` (доход/траты/норма) — **новых полей для них не нужно**.
- Проекция net worth и ETA целей считаются **на клиенте** (есть `net_worth_series`,
  `kpi` темп, `useGoals` с target/deadline/balance, `refs.rates` для конверсии).
- Контракт ошибок/auth — без изменений (SPEC-013 §6, §8).

## 7. UI / UX

`cloud/admin/src/routes/DashboardPage.tsx` дорабатывается (не переписывается).

```
┌──────────────────────────────────────────────────────────────────┐
│  Дашборд                       [ Свободные | Со всеми фондами ]    │  ← тумблер
│  Сводка в EUR. Курсы date-aware. Последний курс: 2026-05-26.       │
├──────────────────────────────────────────────────────────────────┤
│ СЕЙЧАС                                                             │
│ ┌ Net worth ─────┐ ┌ Траты/мес ──┐ ┌ Доход/мес ──┐ ┌ Норма ┐ ┌ Runway ┐
│ │ 5 555 €  ↑ +4% │ │ 2 345 ↓ −3% │ │ 3 100 ↑ +4% │ │ 25% ↑ │ │ 3.1мес │
│ │ ‹спарклайн›    │ │ ‹спарклайн› │ │ ‹спарклайн› │ │‹спарк›│ │‹спарк› │
│ │ свободно,      │ │ все траты   │ │ свободный   │ │ из    │ │ без    │
│ │ без фондов     │ │ за 3 мес    │ │ доход       │ │своб.  │ │ фондов │
│ └────────────────┘ └─────────────┘ └─────────────┘ └───────┘ └────────┘
├──────────────────────────────────────────────────────────────────┤
│  Net worth по времени            [ total · форма · валюта ]        │
│  [сплошная линия — факт; пунктир — проекция на 6–12 мес вперёд]    │
├──────────────────────────────────────────────────────────────────┤
│  Цели — прогноз достижения                                        │
│  🏠 Ипотека   9 000 / 40 000 € · +1000 €/мес → к янв 2030          │
│               дедлайн июн 2028 · ⚠ опаздываешь на ~19 мес          │
└──────────────────────────────────────────────────────────────────┘
```

- **Тумблер**: компактный segmented control справа от заголовка (стиль period-toggle
  на дашборде). State в компоненте, дефолт `free`.
- **KPI-карточка**: добавляются (1) Δ-бейдж справа от цифры (↑/↓ + %, цвет по
  смыслу: для дохода/нормы/net worth рост = positive, для трат рост = negative),
  (2) спарклайн (тонкая линия, ~40px высотой, без осей, brand-цвет), (3) подпись-линза.
- **Δ цвет/знак**: «хорошо» зелёным, «плохо» — приглушённым/destructive. Скрыт, если
  предыдущее окно пустое (E2).
- **Проекция**: на net worth chart — продолжение линии пунктиром, чуть бледнее,
  с подписью конечной точки. Только для линии total/free; для stacked area
  (форма/валюта) проекцию не рисуем.
- **ETA целей**: новый блок-карточка под графиками. Каждая активная цель: прогресс
  (balance/target), темп, ETA-дата, сравнение с дедлайном (бейдж успеваю/опаздываю).
  Цели без target → «нет суммы». Достигнутые → «достигнута».
- Состояния loading/empty/error — как в SPEC-013, расширяются на новые блоки.

## 8. Security

- Без изменений к SPEC-013 §8: read-only, JWT Bearer, generic 5xx, без утечек,
  суммы не логируем. Новые поля — те же чувствительные финданные за тем же auth.

## 9. Acceptance criteria

- [ ] **AC1**: Тумблер по умолчанию «Свободные»; Net worth = free, Доход = free,
  Норма = free, Runway = free. Переключение на «Со всеми фондами» меняет эти 4 на
  total-версии **без рефетча**. Траты не меняются.
- [ ] **AC2**: Под каждым KPI — подпись, корректная для текущей линзы.
- [ ] **AC3**: Δ-бейдж на каждой KPI = изменение к предыдущему 3-мес окну (net
  worth — к значению 3 мес назад); знак/цвет по смыслу; скрыт при пустом prev.
- [ ] **AC4**: `monthly_income_free = monthly_income` когда в окне нет goal-доходов
  (случай Stepan'а); при наличии goal-дохода — строго меньше на их сумму.
- [ ] **AC5**: Спарклайн в каждой карточке отражает соответствующую series (net
  worth / income / expense / savings rate помесячно); пустые данные → без падения.
- [ ] **AC6**: Проекция net worth — пунктир вперёд по темпу `income_free − burn`;
  при темпе ≤ 0 не растёт; рисуется только для линии (не stacked area).
- [ ] **AC7**: ETA целей: при темпе > 0 и target > balance — дата достижения +
  сравнение с дедлайном; темп ≤ 0 → «не достигается»; balance ≥ target → «достигнута».
- [ ] **AC8**: `net_worth_eur`/`free_net_worth_eur` (и их prev) консистентны: free =
  total − targeted; AC2 SPEC-013 (кросс-чек с /accounts) не сломан.
- [ ] **AC9**: 401/5xx/empty — поведение SPEC-013 сохранено; новые блоки не роняют
  страницу при отсутствии данных.
- [ ] **AC10**: tsc + build зелёные; `test_admin_ui.py` снимает дашборд в обеих
  линзах (light/dark), визуально проверено.

## 10. Test plan

- **Worker**: smoke `GET /v1/web/dashboard` — новые поля присутствуют, типы верны;
  `monthly_income_free` = `monthly_income` на данных Stepan'а (нет goal-доходов в
  окне), и строго меньше на синтетике с goal-доходом; prev-окно сдвинуто корректно.
- **Admin SPA**: `test_admin_ui.py` — расширить mock dashboard (новые kpi-поля +
  цель с target/deadline для ETA); сценарии: дефолтная линза, переключение тумблера,
  скрин обеих линз, ETA-блок. Скриншоты light/dark.
- **Расчёт ETA вручную**: remaining/темп сверить с подписью.
- **Regression**: `/accounts`, `/incomes`, `/goals`, остальной дашборд (графики,
  фильтры) — не затронуты; Mini App — не затронут.

## 11. Risks & open questions

- **R1 (линза дохода неинтуитивна)**: «свободный доход» = доход минус отложенное в
  цели. Митигировано подписью под KPI + тем, что дефолт совпадает с интуицией
  («сколько я свободно зарабатываю»).
- **R2 (проекция вводит в заблуждение)**: линейная экстраполяция — груба. Митигация:
  пунктир (явно «прогноз, не факт») + подпись «при текущем темпе».
- **R3 (ETA в target_currency vs темп в EUR)**: remaining в target_currency → EUR по
  latest rate; темп в EUR. Допустимо для прогноза (не историческая точность).
- **D1 (решено 2026-05-26)**: дефолтная линза — «Свободные деньги» (по просьбе
  Stepan'а). Тумблер не персистится (NG6).
- **D2 (решено)**: спарклайны и проекция/ETA — на клиенте из существующих series +
  goals, чтобы минимизировать worker-изменения (только free/prev KPI-поля).
- **OQ1**: проекция — на сколько месяцев вперёд по умолчанию (6 или 12)? Предв. 12,
  совпадает с окном истории. Уточнить при реализации/ревью.
