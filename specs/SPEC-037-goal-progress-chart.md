# Web Admin — график прогресса цели на /goals/:id

## 1. Context & Problem

На странице цели `/goals/:id` сейчас есть карточка с прогресс-баром (баланс/таргет)
и таблица «История пополнений» — но **нет визуализации динамики во времени**.
Не видно, как цель копилась по месяцам, успеваем ли к дедлайну и когда реально
закроем при текущем темпе. Это последнее место в Admin, оставшееся «только
таблицей» (post-mvp-roadmap 2.5, NG2 SPEC-018). Все данные уже есть: `getGoalDetail`
возвращает вклады с date-aware `delta_in_target` (поток, ADR-020/SPEC-025), а
дашборд уже умеет line-чарты с пунктирным прогнозом (ECharts).

## 2. Goals

- G1: На `/goals/:id` под карточкой цели — новая карточка «Прогресс» с line-чартом
  накопленного баланса цели **по месяцам** (в `target_currency`).
- G2: Горизонтальная линия **target** (`target_amount`), если цель с таргетом.
- G3: Вертикальный маркер **дедлайна** (`deadline`), если задан.
- G4: **ETA-прогноз** — пунктирная линия от текущего баланса к target по среднему
  темпу последних месяцев; доходит до target в прогнозный месяц завершения.
  Пунктир — потому что это forecast (memory `dashed-line-means-forecast`,
  ровно как «Прогноз» net worth на дашборде).
- G5: Чисто клиентский расчёт поверх уже загруженных `contributions`
  (`delta_in_target` + `date`). **Никаких новых endpoint'ов, Worker-кода, миграций.**
- G6: Тема/цвета через общий `chartTheme()`; линия факта — в цвете цели (`goal.color`).
  Light/dark корректны.

## 3. Non-Goals

- NG1: Worker-эндпоинт / SQL-серия для прогресса. Вклады уже приходят целиком в
  `getGoalDetail`; cumulative по месяцам считается в браузере (как Mini App stats /
  dashboard-серии на клиенте). При росте до тысяч вкладов на цель — пересмотреть
  (тот же потолок, что у других клиентских агрегаций).
- NG2: Изменение математики баланса цели. Берём готовый `delta_in_target`
  (зафиксирован по курсу даты вклада, ADR-020) — не пересчитываем, не реконструируем
  историч. FX. Финальный cumulative обязан совпасть с `goal.balance` (R1).
- NG3: Реконструкция исторического mark-to-market в EUR. График — в `target_currency`
  (как прогресс-бар на той же странице), не в EUR. EUR-сводки остаются на дашборде.
- NG4: Распределение свободного потока между несколькими целями / сравнение целей
  (это M3, Фаза 3). Здесь — одна цель.
- NG5: Интерактивное редактирование из графика (drill/прыжок к вкладу). Таблица ниже
  уже даёт CRUD. График — read-only визуализация.
- NG6: Новая чарт-библиотека. Используем ECharts (`echarts-for-react`) — уже
  зависимость Admin, и dashed-forecast паттерн уже отлажен на дашборде.

## 4. User journeys

### Happy path — цель с таргетом и дедлайном

1. Пользователь открывает `/goals/:id` активной цели (есть валюта, target, deadline,
   несколько пополнений).
2. Под карточкой цели — карточка «Прогресс» с графиком:
   - сплошная линия в цвете цели: накопленный баланс по месяцам (факт);
   - горизонтальная линия target;
   - вертикальный пунктир-маркер дедлайна;
   - пунктирная линия ETA от последней фактической точки к target.
3. По наведению — tooltip с месяцем и суммой.
4. Видит: дойдёт ли пунктир до target раньше или позже маркера дедлайна.

### Happy path — открытая цель (без target)

1. Цель без `target_amount` — график показывает только сплошную линию накопления
   (без target-линии и ETA). Полезно «сколько уже отложено и как растёт».

### Edge cases

- E1: Нет target → только линия факта, без target-линии и ETA (E1).
- E2: Нет пополнений (`contributions` пуст) → карточку графика **не показываем**
  (таблица уже даёт свой empty-state «Пополнений пока нет»).
- E3: Валюта цели не задана (`needsCurrency`) → график не показываем (на странице уже
  amber-предупреждение «Валюта цели не задана»; cumulative в target_currency невозможен).
- E4: Один месяц данных → одна точка (ECharts показывает symbol для единственной точки).
- E5: Цель достигнута (`balance ≥ target`) → ETA не рисуем (уже дошли); линия факта
  может лежать на/над target-линией — это и есть «достигнуто».
- E6: Средний темп ≤ 0 (нет вкладов в последних месяцах / отток) → ETA не рисуем.
- E7: Дедлайн в прошлом → маркер всё равно показываем (визуальный сигнал просрочки);
  ETA, если рисуется, может уйти правее маркера.
- E8: `delta_in_target == null` у части вкладов (нет курса) → в cumulative считаем как 0
  (как и `goal.balance`); число «N без курса» уже показано в шапке цели.

## 5. Data model

**Без изменений.** Никаких миграций/таблиц/колонок. Источник — те же
`contributions` (поле `delta_in_target` в `target_currency`, date-aware) и поля
`goal` (`target_amount`, `target_currency`, `deadline`, `balance`), которые уже
возвращает `GET /v1/web/goals/:id` (`getGoalDetail`). Расчёт — в браузере.

## 6. API contract

**Без новых endpoint'ов.** Используется существующий `GET /v1/web/goals/:id`
(через `useGoalDetail`) под JWT (Bearer). Worker не трогаем.

## 7. UI / UX

Новая карточка «Прогресс» (компонент `components/GoalProgressChart.tsx`),
встраивается в `GoalDetailPage` между карточкой цели и карточкой «История
пополнений». Рендерится только когда `ccy` задан и `contributions.length > 0`.

### Тонкости
- **Cumulative по месяцам.** Бакетируем `delta_in_target` по месяцу вклада, затем
  prefix-sum → накопленный баланс на конец месяца; месяцы без вкладов несут предыдущее
  значение (forward-fill, линия идёт горизонтально). Последняя точка == `goal.balance` (R1).
- **Диапазон оси X.** От месяца первого вклада до `max(текущий месяц, месяц дедлайна,
  месяц ETA)`.
- **ETA-темп.** Средний месячный приток за последние ≤3 месяца истории. `monthsToTarget
  = ceil((target − balance) / rate)`; ETA-месяц = текущий + monthsToTarget. Пунктир
  стыкуется с последней фактической точкой (массив null-ов до неё, как «Прогноз» дашборда).
- **target / deadline.** Через `series.markLine` (горизонталь на `target_amount`;
  вертикаль на месяце дедлайна) — цвет `chartTheme().muted`, неброско.
- **Forecast = пунктир.** ETA и target/deadline-маркеры — пунктир/тонкие; сплошная
  только у факта (инвариант: пунктир зарезервирован за прогнозом/референсом).
- **Цвета.** Факт — `goal.color`; target/ETA/маркеры — `muted`; через `chartTheme()`,
  переживает light/dark. Высота ~260-280px (как dashboard-чарты).
- **Tooltip.** trigger axis, формат суммы в `target_currency`.

## 8. Security

- Чтение под тем же Google-OAuth→JWT, что и вся страница цели (Worker не меняется,
  новых поверхностей нет).
- Никаких новых данных в логи/телеметрию — всё в браузере. XSS: имена/числа —
  React-children / ECharts data (не innerHTML).

## 9. Acceptance criteria

- [x] AC1: На `/goals/:id` цели с валютой и ≥1 пополнением под карточкой цели видна
      карточка «Прогресс» с line-чартом накопленного баланса по месяцам.
- [x] AC2: Последняя точка линии факта == `goal.balance` (совпадение с прогресс-баром, R1).
- [x] AC3: При наличии `target_amount` — горизонтальная линия target на графике.
- [x] AC4: При наличии `deadline` — вертикальный маркер дедлайна.
- [x] AC5: При незавершённой цели с положительным средним темпом — пунктирная ETA-линия
      от факта к target; при темпе ≤0 или достигнутой цели ETA не рисуется.
- [x] AC6: Цель без target → только линия факта (без target/ETA). Цель без валюты или
      без пополнений → карточка графика не рендерится.
- [x] AC7: Light/dark корректны (цвета через `chartTheme()`); линия факта в цвете цели.
- [x] AC8: `tsc --noEmit` (admin) зелёный; Worker/D1 не тронуты (diff только в
      `cloud/admin/src`).
- [x] AC9: Playwright Admin-харнес рендерит страницу цели с графиком без console-ошибок
      (light+dark).

## 11. Risks & open questions

- R1: cumulative-сумма последней точки обязана == `goal.balance` (оба = Σ delta_in_target,
  null исключён). Проверить на моке и реальной цели.
- R2: ETA — грубая линейная экстраполяция по среднему темпу; это оценка, помечена
  пунктиром. Не претендует на точность (нерегулярные вклады). Приемлемо для сигнала.
- R3: Перф — cumulative O(вкладов + месяцев) под `useMemo`; в пределах NG1.
- OQ1 (решено owner'ом 2026-06-27): берём 2.5 как следующую фичу.

## 12. Out of scope для review

- График в `target_currency`, не в EUR (NG3) — сознательно, паритет с прогресс-баром
  цели. Не ругать за «нет EUR-сводки» (она на дашборде).
- Линейная ETA вместо сложной модели (NG4/R2) — сознательно.
