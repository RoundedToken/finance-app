# Фикс — двойной учёт импортированной зарплаты в историческом net worth

> Фидбэк owner (2026-06-22): «график net когда выбираю "Со всеми фондами", на исторических данных
> показывает в марте и апреле цифры, которых у меня тогда не было — поломка в датах или ещё чего».

## 1. Context & Problem

Исторический ряд net worth строится `dashboard.ts:balanceAt(bucket, конец_месяца)` =
последний manual-снапшот `≤ T` (baseline) **+ события после него**. «После» при совпадении даты
решается по `created_at` (SPEC-024, `ledger.ts:55-58`):

```
событие учитывается, если  event.date > baselineDate
                       ИЛИ (event.date == baselineDate И event.createdAt > baselineCreatedAt)
```

**Баг в данных легаси-импорта.** При импорте из `Finances.xlsx` (25.05.2026) записались:

- **198 снапшотов** с `created_at = 2026-05-25T17:23:42Z` (note = `импорт из Finances.xlsx`);
- **20 доходов «зарплата (legacy)»** с `created_at = 2026-05-25T19:09:32Z` — на ~1.5 часа **позже**.

Зарплата падала на `eur-cash` 15-го числа — **в ту же дату, что и снапшот** этого ведра. Снапшот
15-го уже отражает полученную в этот день зарплату (это замер баланса). Но из-за tie-break
(`19:09 > 17:23`) доход считается «случившимся после снапшота» и **прибавляется поверх** → двойной
учёт:

```
eur-cash, конец марта:  снапшот 2026-03-15 = 6040  +  доход 2026-03-15 = 2330  =  8370  (фантом)
```

**Масштаб** (offline-реконструкция по дампу D1 от 2026-06-09):

| Месяц | Было (баг) | Должно быть | Фантом |
|---|---|---|---|
| 2025-09 … 2026-01 | 12457…13121 | 10457…11121 | +2000/мес |
| 2026-02 | 13643 | 11313 | +2330 |
| 2026-03 | 14369 | 12039 | +2330 |
| 2026-04 | 13750 | 11420 | +2330 |
| 2026-05 | 9752 | 9752 | 0 |
| 2026-06 (текущее) | 9550 | 9550 | 0 |

Завышен **весь хвост истории за последний год** (baseline почти всегда — снапшот на 15-е, на 15-е —
зарплата). Текущее значение (ручные снапшоты) — корректно. Всего по `eur-cash` — 19 коллизий-доходов
на +40 990 € (+ 4 импортированных расхода в минус, тот же класс).

**Почему видно в «Со всеми фондами», а не в «Свободные».** В линзе «Свободные»
(`DashboardPage.tsx:332-333`) из ряда вычитается константный `shift = targeted + invested` и результат
клампится в `max(0, …)` — фантом тонет. В «Со всеми фондами» (`lens="total"`, `shift=0`) рисуется
сырой `Σ by_bucket` — двойной счёт виден напрямую. Сам баг в обеих линзах.

## 2. Goals

- **G1**: исторический net worth по импортированным месяцам = реальные снапшоты, без +2330 фантома.
- **G2**: live-поведение SPEC-024 (событие после ручного снапшота того же дня — учитывается) **сохранено**.
- **G3**: фикс точечный, идемпотентный, обратимый (исходный `created_at` известен).

## 3. Non-Goals

- **NG1**: не меняем tie-break в коде (`ledger.ts`). Глобальное правило «снапшот поглощает все события
  своей даты» ломает май (−127): там есть **легитимное** событие после ручного снапшота того же дня —
  ровно тот live-кейс, ради которого SPEC-024 сделан. Корень — в данных, не в логике.
- **NG2**: не трогаем live-снапшоты (ручной ввод, `created_at` в формате со space) и cashflow-график
  (он читает события напрямую, не через `balanceAt`).

## 4. Solution

Миграция `0017_fix_imported_snapshot_created_at.sql` поднимает `created_at` у 198 импортированных
снапшотов в конец дня импорта, чтобы импортированные события той же даты сортировались **до** снапшота
(снапшот = ground truth дня → поглощает свои события):

```sql
UPDATE snapshots
   SET created_at = '2026-05-25T23:59:59Z', updated_at = datetime('now')
 WHERE source = 'manual'
   AND created_at = '2026-05-25T17:23:42Z'   -- точная метка импорт-батча (198 строк)
   AND deleted_at IS NULL;
```

Значение `23:59:59Z` гарантированно после всех импортированных T-format событий (макс. `19:09:32Z`),
в том же T/Z-формате (корректное строковое сравнение с импортированными доходами). Live-снапшоты имеют
`created_at` в формате со space и другую дату — не затрагиваются.

Применение (memory `d1-migrations-apply-via-execute-file`): **backup → `wrangler d1 execute --file`**,
не `migrations apply`. Код Worker/Pages не меняется — деплой не требуется, только D1 UPDATE.

## 5. Acceptance Criteria

- **AC1**: после миграции `eur-cash` на конец 2026-03 = 6040 €, 2026-04 = 6455 € (реальные снапшоты),
  не 8370/8785. Проверено offline на копии дампа.
- **AC2**: net worth за 2026-05 и текущий (2026-06) — без изменений (9752 / 9550).
- **AC3**: затронуто ровно 198 строк (`changes() = 198`), все — `note = импорт из Finances.xlsx`.
- **AC4**: повторный прогон миграции безопасен (idempotent: тот же `created_at`).
- **AC5**: «Со всеми фондами» на проде — историческая линия net worth опустилась на величину фантома,
  совпадает с offline-реконструкцией.
