# Фикс — итог дня на главной не сходится со списком

> Фидбэк owner (2026-06-22, скриншот): «итоговая сумма подсчёта большая, но не показывает все
> транзакции за сегодня — это баг». На скрине итог «СЕГОДНЯ» = 21 660 RSD, а видно 6 трат на 9 929 RSD.

## 1. Context & Problem

На главной Mini App блок «недавние дни» (`MainScreen.tsx:RecentDays`) для каждого из 2 последних дней
показывает итог `DayTotal` **по всем** тратам дня, но сам список рендерит только первые 6
(`rows.slice(0, 6)`). Если за день >6 трат — итог больше, чем сумма видимых строк → выглядит как
ошибка подсчёта.

**Диагностика на проде (2026-06-22):** за сегодня 16 трат RSD на сумму ровно 21 660 (итог корректен);
bootstrap отдаёт все траты (`limit=20000`), т.е. все 16 на клиенте есть — проблема **только** в
обрезке отображения. `HistoryScreen` обрезки не имеет (показывает все строки). Баг локализован в
`RecentDays`. Итог `DayTotal` и конверсия EUR — корректны, не трогаем.

## 2. Goals

- **G1**: список трат дня на главной показывает **все** траты дня → сумма видимых строк сходится с
  итогом `DayTotal`.
- **G2**: правка/удаление любой траты дня доступны прямо с главной (раньше >6-я строка была недоступна
  для swipe/edit без перехода в Историю).
- **G3**: фикс точечный, только Mini App (Worker/D1 не трогаем).

## 3. Non-Goals

- **NG1**: не меняем `DayTotal` (итог корректен) и canonical EUR-конверсию (SPEC-016).
- **NG2**: не меняем «2 последних дня» на главной — это осознанный preview; полная история по дням —
  в `HistoryScreen`.
- **NG3**: виртуализация длинных списков — отдельный backlog-пункт (3.4), не для этого фикса (single-user,
  десятки строк/день рендерятся мгновенно, как в Истории).

## 4. Solution

`MainScreen.tsx:RecentDays` — убрать обрезку `rows.slice(0, 6)` → рендерить `rows` целиком. Тогда
видимый список = множество, по которому считается `DayTotal`. (Альтернатива «6 + ещё N → История»
отвергнута: для finance-итога честнее видеть все строки, чем объяснять расхождение; History остаётся
для кросс-дневного просмотра.)

## 5. Acceptance criteria

- [ ] AC1: за день с N трат на главной отображаются **все N** строк (не 6).
- [ ] AC2: сумма видимых строк дня сходится с итогом `DayTotal` (по нативной валюте) — для 16 трат
  сегодня итог 21 660 RSD совпадает с суммой строк.
- [ ] AC3: 7-я+ трата дня доступна для tap-edit / swipe-delete прямо с главной.
- [ ] AC4: дни с ≤6 трат и пустой стейт («Пока нет трат») — без регрессии.

## 6. Test plan

- **Mini App (Playwright, light/dark)**: сид >6 трат на сегодня → главная рендерит все строки; сумма
  отрисованных строк == отображаемому итогу дня. tsc + vite build.
- **Прод-верификация**: открыть Mini App — «СЕГОДНЯ» показывает все 16 трат, итог сходится.
