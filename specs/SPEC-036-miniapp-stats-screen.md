# Mini App — экран «📊 Статистика»

## 1. Context & Problem

В старом vanilla Mini App (SPEC-003) была полноценная вкладка «📊 Статистика»:
KPI + donut по категориям + список категорий + trend chart + drill-down. При
React-rewrite (SPEC-014) аналитика **сознательно не воссоздана** — scope Mini App
сузили до «только ввод», аналитика ушла в Web Admin (CLAUDE.md правило 11 +
post-mvp non-goal). Web Admin-дашборд хорош на десктопе, но требует перехода в
браузер + Google-вход и не «живёт в Telegram», где происходит ввод.

Owner-решение (2026-06-27): **разморозить scope и вернуть статистику в Mini App**,
чтобы «куда уходят деньги за период» было видно одним тапом из той же Mini App,
без браузера. Решение фиксируется ADR-021 (отменяет freeze, заданный ADR-012 +
правилом 11). Web Admin остаётся домом «глубокой» аналитики (портфель, цели,
дашборд, net worth) — Mini App-статистика читает **только траты**, она не
дублирует и не вытесняет Admin.

## 2. Goals

- G1: Экран «📊 Статистика» открывается из меню Mini App (кнопка «Статистика»
  сейчас disabled со словом «скоро» — её включаем). По умолчанию — текущий месяц.
- G2: Period picker **Месяц / Год / Всё** + навигация ‹ › по периодам — точно как
  в Истории (SPEC-035), чтобы UX/код переиспользовались (общая модель
  `{mode, year, month}`, общие границы `bounds`, та же логика disabled prev/next).
- G3: KPI: итог за период (EUR) + ≈ средн./день + кол-во трат + дельта Δ% к
  прошлому периоду (`▲/▼/→/▲ new`).
- G4: Donut: топ-8 категорий по сумме + «Прочее»; tap по сегменту → drill-down.
- G5: Список категорий: процент + абсолют (EUR) + цветной bar-индикатор; tap →
  drill-down. Цвет сегмента donut == цвет точки/бара категории (один палитра-маппинг).
- G6: Trend chart: бары по дням (Месяц) или по месяцам (Год/Всё) + средняя линия +
  читабельная ось X; «сегодняшний» бар/месяц выделен.
- G7: Drill-down: tap по категории → модал со списком трат категории за период;
  tap по трате → существующий экран редактирования (`loadEdit`).
- G8: Чисто клиентский расчёт поверх уже загруженных трат (`useExpenses`,
  тот же набор что у Истории). **Никаких новых endpoint'ов, Worker-кода и миграций D1.**

## 3. Non-Goals

- NG1: Переключатель базовой валюты. В React baseCurrency мёртв (Settings-модалки
  нет, `s.baseCurrency` всегда EUR). Статистика считается и показывается в **EUR**
  через `amount_eur` (date-aware, SPEC-016) — это согласовано с Историей и с ADR-014
  (расход = поток по курсу своей даты). Сознательное отличие от SPEC-003, который
  пересчитывал по последнему курсу в выбранную базу. Возврат baseCurrency — отдельная
  фича, если когда-нибудь понадобится.
- NG2: Период «Неделя» (был в SPEC-003). Отброшен ради консистентности с Историей
  (SPEC-035 — 3 режима). Можно добавить позже как хвост, не блокер.
- NG3: Worker-side агрегация / SQL для статистики. ~2000 трат в памяти WebView —
  норма (SPEC-003 NG5, R4). Потолок тот же: при 10k+ — серверная агрегация (отдельный
  триггер, как виртуализация SPEC-034).
- NG4: Аналитика по доходам / снапшотам / портфелю / net worth. Это остаётся **только**
  в Web Admin (глубина). Mini App-статистика — про расходы и только расходы.
- NG5: Экспорт (CSV/PDF/share). Системный скриншот Telegram достаточно (SPEC-003 NG4).
- NG6: ECharts / тяжёлые чарт-библиотеки. Donut — рукотворный SVG, trend — CSS-бары
  (порт vanilla). Mini App-бандл остаётся лёгким для мобильного WebView.
- NG7: Любой новый CRUD в Mini App. Единственная мутация из статистики — переход в
  **уже существующий** edit-экран траты через drill-down (это «ввод», он и так разрешён).

## 4. User journeys

### Happy path — посмотреть статистику за месяц

1. Пользователь в Mini App тапает `☰` (меню) → «📊 Статистика».
2. Открывается экран, по умолчанию — **текущий месяц**, offset на текущем периоде.
3. Сверху: заголовок «Статистика» + кнопка назад; сегмент `[Месяц][Год][Всё]`;
   степпер `‹ Июнь 2026 ›`.
4. KPI: `1 234 € · ▲ 12%`, строкой ниже `≈ 49 €/день · 142 траты`.
5. Donut: топ-8 категорий + «Прочее» в центре — итог EUR.
6. Список категорий: `🍔 Еда 42% · 518 €` с цветным баром; ниже остальные по убыванию.
7. Trend: бары по дням месяца, средняя линия, ось `1 · 7 · 14 · 21 · 28`.

### Happy path — drill-down в категорию

1. Тап по сегменту donut **или** по строке категории «Еда».
2. Открывается модал: заголовок «Еда», мета `Июнь 2026 · 23 траты · 518 €`,
   список трат категории за период (новые сверху).
3. Тап по трате → модал закрывается, открывается экран редактирования траты
   (как из Истории) — можно поправить/удалить.

### Happy path — сменить период

1. Тап `Год` → бары становятся помесячными, KPI/donut/список пересчитываются за год.
2. `‹` уводит в прошлый год; `›` (в будущее) задизейблен на текущем периоде.
3. `Всё` → диапазон min..max дат трат; навигация `‹ ›` скрыта.

### Edge cases

- E1: У части трат `amount_eur == null` (нет курса на дату). Такие траты считаются
  в `count`, но **исключаются** из сумм/donut/trend; в KPI показывается
  `! N без курса` (danger). (= SPEC-003 E1.)
- E2: Период пуст (нет трат) — KPI: «Трат в этом периоде нет»; donut/список/trend
  молчат («Нет данных»). (= SPEC-003 E2.)
- E3: «Всё время» при полном отсутствии трат — диапазон сегодня..сегодня, пустые состояния.
- E4: `prev.total == 0`, `cur.total > 0` — дельта `▲ new`. (= SPEC-003 E4.)
- E5: Длинное total-число в центре donut — адаптивный font-size (24/21/17/13 по длине).
- E6: Trend из одной точки (1 день в «Всё» / 1 месяц в «Год») — один бар, один тик оси.
- E7: Navigation bounds — не листаем раньше самого раннего месяца/года с тратами и
  не в будущее (переиспользуем `bounds` из Истории).

## 5. Data model

**Без изменений.** Никаких миграций, новых таблиц/колонок. Источник — те же
`expenses` (с `amount_eur`, date-aware EUR из SPEC-016) и `categories`, что уже
загружены `useExpenses()` / `useBootstrap()`. Расчёт — в браузере.

## 6. API contract

**Без новых endpoint'ов.** Используются существующие:
- `GET /v1/expenses?limit=5000` (через `useExpenses`) — полный список с `amount_eur`.
- `GET /v1/bootstrap` (через `useBootstrap`) — `categories` для имён/эмодзи/цветов.

Оба уже под Telegram `initData` HMAC. Worker не трогаем.

## 7. UI / UX

Экран `stats` (новый компонент `screens/StatsScreen.tsx`), регистрируется в
`App.tsx` (`{s.screen === "stats" && <StatsScreen/>}`). Шапка периода — копия
паттерна Истории (segment + stepper) для визуальной консистентности.

```
┌──────────────────────────────────────┐
│ ‹  Статистика                         │  back + title
├──────────────────────────────────────┤
│   [Месяц*] [Год] [Всё]                │  сегмент (как История)
│   ‹  Июнь 2026  ›          1 234 €    │  степпер + итог периода
├──────────────────────────────────────┤

│   ≈ 49 🇪🇺/день · 142 траты            │  meta (+ «! N без курса» если есть)

│            ╭─┤ 1234 ├─╮               │  donut top-8 + Прочее

│  ━━━━━━━━━━━━━━━━━━━━━                 │  bar-fill (цвет = цвет сегмента)

│  1 . . 7 . . 14 . . 21 . . 28          │  ось (5 тиков)
└──────────────────────────────────────┘

drill-down (модал):
┌──────────────────────────────────────┐
```

### Тонкости (порт из SPEC-003, адаптация под React/EUR)
- **Метрика — `amount_eur`.** Агрегация Σ amount_eur; null → в `count`, не в сумму.
- **Палитра.** 12-цветная hue-разнесённая `CHART_PALETTE` (как vanilla), топ-8
  получают уникальный цвет, хвост — `CHART_TAIL`; donut «Прочее» — `CHART_OTHER`.
  Один маппинг `colorByCat` шарится donut'ом, списком и баром.
- **Donut.** SVG `<circle>` + `stroke-dasharray`/`offset`, gap между сегментами,
  адаптивный font центра, stagger fade-in. Tap по сегменту (кроме «Прочее») →
  drill-down. В центре — код валюты `EUR` (color-emoji в SVG `<text>` Telegram
  WebView не рендерит → флаг только в HTML, не в SVG).
- **Trend.** Месяц → бин по дням; Год/Всё → по месяцам; «сегодня»-бар выделен;
  средняя линия = total / elapsed-bins; ось: Месяц — 5 тиков, Год/Всё — до 6
  (для «Всё» label = `май '25`, год обязателен).
- **avg/день.** Для текущего (in-progress) периода делим на **прошедшие** дни, не
  на полную длину периода (иначе средняя занижена в начале месяца).
- **Drill-down → edit.** Закрываем модал, затем `d({ t: "loadEdit", e })` (existing).
- **Haptics.** `haptic("light")` на смену периода/таба, как везде в Mini App.

## 8. Security

- Чтение под тем же Telegram `initData` HMAC, что и весь Mini App. Новых
  auth-поверхностей нет (Worker не меняется).
- Никаких новых данных в логи/телеметрию — всё считается в браузере, ничего не
  отправляется. PII (суммы) не покидают клиент.
- XSS: имена/эмодзи категорий рендерим как React-children (auto-escape), не
  `dangerouslySetInnerHTML`. (Vanilla использовал `innerHTML` + `escapeHtml`; React
  снимает этот класс рисков.)

## 11. Risks & open questions

- R1: Расхождение «итог Истории» vs «итог Статистики» за тот же период должно быть
  нулевым (обе считают Σ amount_eur по тому же клиентскому набору). Проверить, что
  фильтр периода идентичен (date-prefix) — переиспользуем одну логику.
- R2: Производительность: пересчёт агрегатов на каждый ререндер. Митигируем
  `useMemo` по `[expenses, mode, year, month]` (как История).
- R3: «Прочее» в donut кликабельно визуально, но drill-down по нему не делаем
  (нет одной категории) — сегмент «Прочее» не реагирует на tap (как vanilla).
- OQ1 (решено owner'ом 2026-06-27): разморозка scope подтверждена → ADR-021.

## 12. Out of scope для review

- Отсутствие переключателя базовой валюты (NG1) и периода «Неделя» (NG2) —
  сознательно. Не ругать за «фича-паритет» с vanilla SPEC-003.
- Рукотворный SVG/CSS-чарт вместо общей чарт-библиотеки (NG6) — сознательно ради
  лёгкого мобильного бандла.
