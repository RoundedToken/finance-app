# Mini App История — фильтр по периоду

## 1. Context & Problem

Экран «История» в Mini App показывает все траты разом (одним длинным скроллом, сгруппированы по дням). На реальных данных (сотни дней) это неудобно: нельзя быстро посмотреть «сколько за этот месяц» или «за год». Owner (Stepan) попросил разбивку по периодам с удобным переключением. Это улучшение **существующего read-only экрана** (навигация + итоги), не новый CRUD-раздел и не аналитика с графиками → в рамках scope Mini App (CLAUDE.md правило 11; owner-решение 2026-06-26).

## 2. Goals

- G1: По умолчанию История открывается на **текущем месяце**.
- G2: Сегмент-переключатель **Месяц / Год / Всё**; «Всё» = текущее поведение (вся история).
- G3: В режимах Месяц/Год — удобная навигация стрелками «‹ предыдущий / следующий ›» (по месяцам / по годам).
- G4: Итог за выбранный период — **EUR-эквивалент** (`≈ X €`) + счётчик «N трат». Read-only, без графиков/разбивки по категориям.
- G5: Фильтрация — **клиентская**, поверх уже загруженного набора (`useExpenses`). Никаких изменений Worker/D1/контрактов/денег-математики. Виртуализация (SPEC-034) сохранена.
- G6: Красивый, консистентный с приложением UI (сегмент в стиле iOS, haptic на переключении, аккуратные пустые состояния). Временный счётчик `DOM N/M` из SPEC-034 удалён.

## 3. Non-Goals

- NG1: Разбивка по категориям / графики / донаты за период — это аналитика, живёт в Web Admin (правило 11). Owner подтвердил «навигация + итоги».
- NG2: Серверная фильтрация / новые query-параметры / пагинация. Фильтр чисто клиентский (данные уже в памяти).
- NG3: Произвольный диапазон дат (custom range), как «Период» в Admin. Только Месяц/Год/Всё.
- NG4: Сохранение выбранного периода между открытиями экрана. Каждое открытие → текущий месяц (эфемерное состояние экрана).
- NG5: Свайп влево/вправо для смены месяца (жест) — конфликт со SwipeRow удаления. Только стрелки/тапы.
- NG6: Изменение MainScreen.RecentDays (последние 2 дня на главной) — не трогаем.

## 4. User journeys

### Happy path
1. Открываю «История» → вижу сегмент **[Месяц | Год | Всё]** (активен «Месяц»), под ним «‹ Июнь 2026 ›» и справа «≈ X € · N трат». Ниже — траты текущего месяца, сгруппированы по дням (как раньше).
2. Жму «‹» → перехожу на «Май 2026», список и итог обновляются, скролл наверх.
3. Жму сегмент «Год» → «‹ 2026 ›», вижу все траты года + годовой итог.
4. Жму «Всё» → вся история (текущее поведение), итог за всё время.

### Edge cases
- E1: В текущем месяце ещё нет трат → пустое состояние «В этом месяце трат нет» (сегмент/навигация остаются). Owner просил дефолт = текущий месяц, поэтому пустой месяц — валидный экран.
- E2: Навигация ограничена данными: «‹» неактивна раньше самого раннего месяца/года с тратами; «›» неактивна позже текущего (реального) месяца/года — чтобы не листать в пустоту.
- E3: Вообще нет трат → «Пока нет трат», обе стрелки неактивны.
- E4: Переключение Год→Месяц сохраняет год (если это текущий год → текущий месяц, иначе декабрь того года); Всё→Месяц → текущий месяц.
- E5: Загрузка/ошибка — как сейчас (loading-плейсхолдер; История не имеет error-ветки — пре-существующе).

## 5. Data model

Изменений нет. Фильтр клиентский по полю `expense.date` (`YYYY-MM-DD`, локальная дата, SPEC-024). Итог — сумма `amount_eur` (date-aware с worker, SPEC-016).

## 6. API contract

Без изменений. `GET /v1/expenses?limit=5000` → `{ expenses: Expense[] }` как сейчас. Никаких новых параметров.

## 7. UI / UX

Шапка Истории (sticky сверху) из трёх рядов:

```
┌───────────────────────────────────────────┐
│ ←   История                               │  ряд 1: назад + заголовок
│ ┌─────────────────────────────────────┐   │
│ │  Месяц  │   Год   │   Всё            │   │  ряд 2: сегмент (активный = accent-pill)
│ └─────────────────────────────────────┘   │
│   ‹   Июнь 2026   ›        ≈ 1 234 €       │  ряд 3: степпер + итог
│                              42 траты      │
└───────────────────────────────────────────┘
  24 ИЮНЯ                        989 RSD …      ← список (window-virtualized, SPEC-034)
```

- Сегмент: `bg-secondary-bg` трек, активный пункт `bg-accent text-accent-fg` (iOS-стиль), `active:animate-pop`, `haptic("light")` на переключении.
- Степпер: `ChevronLeft/Right` (lucide), круглые тап-цели, `disabled:opacity-30` на границах (E2). Лейбл: «Июнь 2026» (Месяц) / «2026» (Год) / «Всё время» (Всё).
- Итог справа: `≈ <Amount EUR>` + мелкий «N трат» (склонение трата/траты/трат).
- Ряд 3 присутствует во всех режимах (постоянная высота шапки → стабильный `scrollMargin` window-виртуализатора).
- Пустое состояние периода — по центру, шапка остаётся.
- Виртуализация: `useWindowVirtualizer` (SPEC-034 fix) над отфильтрованными днями; смена периода → `window.scrollTo(0,0)`.

## 8. Security

- Auth не меняется (Telegram initData). Эндпоинт тот же. Новых входов/секретов/логов нет. Чистый клиентский рендер/фильтр.

## 9. Acceptance criteria

- [ ] AC1: При открытии Истории активен «Месяц», показан текущий месяц; список содержит только траты этого месяца.
- [ ] AC2: Сегмент Месяц/Год/Всё переключает фильтр; «Всё» показывает всю историю (как до фичи).
- [ ] AC3: В режиме Месяц «‹/›» сдвигают на соседний месяц (с переносом через год); в режиме Год — на соседний год.
- [ ] AC4: «›» неактивна на текущем (реальном) месяце/году; «‹» неактивна на самом раннем месяце/году с данными; при отсутствии трат обе неактивны.
- [ ] AC5: Итог периода = `≈ Σ amount_eur €` по отфильтрованным тратам; счётчик «N трат» корректен и склоняется.
- [ ] AC6: Группировка по дням, DayTotal по дню, swipe-delete, tap-to-edit — работают внутри выбранного периода (SPEC-033/СПЕК-034 не нарушены).
- [ ] AC7: Пустой период → корректное пустое состояние, навигация остаётся доступной.
- [ ] AC8: Временный счётчик `DOM N/M` (SPEC-034) удалён.
- [ ] AC9: Worker/D1/контракты/деньги-математика не тронуты; typecheck+build зелёные; виртуализация работает (window-scroll).

## 10. Test plan

- **Mini App**: Playwright (`test_miniapp_react.py` + scratch-верификатор) light/dark: дефолт-месяц, переключение Месяц/Год/Всё, навигация стрелки + границы (disabled), итог+счётчик, пустой месяц, виртуализация (DOM ≪ всего), swipe/tap внутри периода.
- **Regression**: MainScreen.RecentDays без изменений; SPEC-033 (DayTotal) и SPEC-034 (windowing) не сломаны.
- **Worker**: не затронут; `npm test` зелёный.

## 11. Risks & open questions

- R1: `scrollMargin` window-виртуализатора зависит от высоты шапки — держим высоту ряда 3 постоянной во всех режимах + re-measure на смене набора.
- R2: Дефолт «текущий месяц» при пустом месяце даёт пустой первый экран — осознанно (owner просил дефолт-месяц), смягчено явным пустым состоянием + быстрым переключением на «Всё».
- R3: Склонение «N трат» — отдельный мелкий хелпер (ru plural).

## 12. Out of scope для review

- Не требовать серверной фильтрации/пагинации (NG2), custom-range (NG3), разбивки по категориям/графиков (NG1).
