# Post-MVP Roadmap

> Составлено 2026-05-29 после финализации MVP. Источник: многоосевой анализ (daily-usability / finance-depth / ai-coach / data-quality / tech-debt / infra-perf), заземлённый в реальном коде + `docs/review-mvp-stage1.md`.
>
> **Принцип:** сначала сделать продукт, которому **можно доверять каждый день** (фидбек, корректность данных, безопасность), потом добавлять **глубину**, и только по реальному триггеру — **продвинутое/масштаб**. Калибровка — single-user, без золочения.

## Цель

Полноценный daily-driver: каждое число на экране либо верно, либо честно помечено приблизительным; каждое действие даёт явный отклик; данные не теряются; аналитика отвечает на «сколько у меня / куда движется / сколько ещё можно».

|---|---|---|---|

Бьёт по latency первого экрана + free-tier CPU **на каждом** входе в Admin. Чистый рефактор путей чтения. |
| 1.9 | **Trust-date бейдж «данные достоверны с YYYY-MM»** | M/S | Левая часть графиков (до baseline/курсов) — реконструкция, не факт. `markArea` + подпись. Снимает когнитивную ловушку «читаю приблизительное как точное». |

**Старт Фазы 1:** 1.1 (toast) → 1.2 (isError) → 1.4 (CI) — всё S, и закрывают доверие + защиту будущей работы. 1.5 (зарплата) — как только дашь RSD-суммы.

## Сознательные non-goals (не делать)

- **AI:** weekly digest, goal check-in, onboarding-опрос + 3-мес re-check, `coach_profile`/`coach_surveys`, мульти-канальная доставка (Mini App push). Для одного юзера, который ежедневно открывает дашборд, — дублирование + золочение (review P4).
- **Worker → Pages Functions** миграция: топология работает, auth/CORS чисты — рефактор без пользовательской ценности и с риском задеть рабочую auth-связку.
- **Mini App расширение** за пределы ввода расходов (заморожено; аналитика — в Admin).
- **Расширение откатанных фич** (chains, goal-tagged tx, receipt scanner) без новой веской причины.
